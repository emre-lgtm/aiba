# Admin CMS Plani — AIBA STONE Tam Icerik Yonetimi

## Ozet
Ana sayfadaki TUM icerigin admin panelinden yonetilmesi: Hero slider, Hizmetler, Site Ayarlari (iletisim bilgileri, nav linkleri, footer). Mevcut Portfolio yonetimi zaten mevcut. Tum hardcoded veriler `constants.ts`'den Supabase'e tasinacak.

## Mevcut Durum
- Portfolio (CRUD) zaten admin'de mevcut ✅
- Hero slaytlar `HERO_SLIDES` sabiti — hardcoded
- Hizmetler `SERVICES` sabiti — hardcoded
- Site ayarlari `SITE` sabiti — hardcoded
- Nav linkleri `NAV_LINKS` sabiti — hardcoded
- Footer'daki adres, telefon, email — hardcoded

## Uygulama Adimlari

### Adim 1 — Supabase migration (yeni tablolar)
- `hero_slides`, `services`, `site_settings` tablolarini olustur
- RLS politikalari ekle
- Baslangic verilerini mevcut hardcoded verilerden insert et
- Storage bucket `hero-images` olustur

### Adim 2 — API rotalari
- Hero CRUD: `/api/hero`, `/api/hero/[id]`
- Services CRUD: `/api/services`, `/api/services/[id]`
- Settings: `/api/settings` (GET/PUT — tek satir)

### Adim 3 — Admin sayfalari ve bilesenleri
- Hero Manager: Slayt listesi + ekleme/düzenleme modal (gorsel yukleme + AI generate)
- Services Manager: Hizmet kartlari + ekleme/düzenleme modal (icon secimi)
- Settings Form: Site ayarlari tek form (nav links, contact, footer)

### Adim 4 — Sidebar guncelleme
- Hero, Services, Settings linklerini ekle

### Adim 5 — Public site entegrasyonu
- `hero.tsx` — `HERO_SLIDES` yerine `/api/hero`'dan cek, fallback ile constants.ts
- `services.tsx` — `SERVICES` yerine `/api/services`'den cek
- `navbar.tsx` — `NAV_LINKS` + `SITE` yerine `/api/settings`'den cek
- `footer.tsx` — hardcoded veriler yerine `/api/settings`'den cek

### Adim 6 — Build ve test
- `npm run lint` + `npm run build`

## Supabase Migration (ek SQL)

```sql
-- Hero slaytlari
CREATE TABLE IF NOT EXISTS hero_slides (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  title text NOT NULL,
  accent text NOT NULL,
  description text NOT NULL,
  image_url text NOT NULL,
  sort_order int DEFAULT 0,
  is_active boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);

-- Hizmetler
CREATE TABLE IF NOT EXISTS services (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  icon text NOT NULL DEFAULT 'Gem',
  title text NOT NULL,
  description text NOT NULL,

-- Site ayarlari (tek satir)

-- RLS

ALTER TABLE services ENABLE ROW LEVEL SECURITY;
ALTER TABLE site_settings ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Public read services" ON services FOR SELECT TO anon USING (true);
CREATE POLICY "Public read settings" ON site_settings FOR SELECT TO anon USING (true);

CREATE POLICY "Auth full hero" ON hero_slides FOR ALL TO authenticated USING (true) WITH CHECK (true);

CREATE POLICY "Auth full settings" ON site_settings FOR ALL TO authenticated USING (true) WITH CHECK (true);
```

## Onemli Notlar
- Mevcut admin sayfalari (portfolio, categories, login) dokunulmuyor
- `constants.ts` fallback olarak kaliyor — API bos donerse veya hata verirse eski hardcoded veriler kullanilir
- Hero slaytlari icin de AI gorsel uretimi entegre edilecek (mevcut `/api/generate-image` rotesi kullanilacak)
- Settings sayfasi tek bir form olacak — nav_links dinamik olarak ekleme/silme/canlandirma destegi
- Icon secimi icin Lucide icon listesinden secim yapilabilecek dropdown
- Gorsel yukleme hem dosya yukleme hem AI uretimi destekleyecek (mevcut pattern)
